# AIsci 演示部署指南 (Demo Guide)

本指南旨在帮助您快速部署并向客户演示 AIsci 智能诊疗辅助系统。

## 1. 快速启动 (Windows)

在项目根目录下，双击运行 **`演示启动.bat`**。

该脚本会自动完成以下操作：
- 检查 Python 环境。
- 自动安装缺失的依赖库。
- 自动初始化 `config.json` 配置文件。
- 启动后端 API 服务器。
- **自动获取您的局域网 IP**（方便移动端或其他电脑接入）。

## 2. 访问演示界面

服务器启动后，控制台会显示访问地址：

- **本机演示**：在浏览器打开 [http://localhost:5000](http://localhost:5000)
- **跨设备演示**：在同一局域网下的平板、手机或其他电脑浏览器输入 `http://[您的局域网IP]:5000`

## 3. 核心演示流程建议

1.  **新建病例**：点击左上角“新建病例”按钮，输入模拟患者信息（姓名、年龄、性别）。
2.  **语音录入**：点击“开始接诊”录音按钮，模拟一段医生与患者的对话。
    - *示例对话*：“你好，张先生。今天哪里不舒服？（患者：我咳嗽三天了，还有点发烧...）最高烧到多少度？（患者：38度5...）”
3.  **实时转录**：展示左侧窗口实时出现的转录文字，体现极速响应。
4.  **结构化分析**：录音停止后，系统自动触发“结构化分析”。
    - 展示“结构化要素”页签：系统自动提取了主诉、现病史、诊断等。
    - 展示“AI 建议”页签：展示系统给出的临床辅助建议。
    - 展示“最终病历报告”页签：展示排版整齐的 Markdown 病历，并演示**手动微调编辑**功能。
5.  **保存与导出**：
    - 点击“保存病例”，演示数据持久化到左侧历史列表。
    - 点击 **Word** 或 **PDF** 导出，实时生成并下载专业的打印版病历。
6.  **历史查阅**：点击左侧历史列表中的条目，演示瞬间加载历史病例。

## 4. 演示前准备清单

- [ ] **API Key 检查**：确保 `config.json` 或环境变量中已配置正确的 LLM API Key（如 DeepSeek）。
- [ ] **麦克风检查**：确保演示机器的麦克风正常开启。
- [ ] **浏览器建议**：建议使用 Chrome 或 Edge 浏览器以获得最佳交互效果。
